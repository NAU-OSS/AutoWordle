# Contributing

Welcome! When contributing to this repository, please first discuss the change you wish to make via GitHub issue or email (ags392@nau.edu). 

## Pull Request Process

1. Ensure any install or build dependencies are removed before the end of the layer when doing a 
   build.
2. Update the README.md with details of changes to the interface, which includes new dependencies.
3. Increase the version numbers in any example files and the README.md to the new version that this
   Pull Request would represent. The versioning scheme we use is [SemVer](http://semver.org/).
4. You may merge the Pull Request once you have the sign-off from Aiden Seay.

## Reporting Bugs

Open a new issue within the GitHub issue tracker and label it as `bug`. Be descriptive and include steps for replicating the bug. Additionally, please write a proposed solution.

## Proposing New Features

Open an issue and label it as `enhancement`. When writing out the new feature, talk about what the feature is about and how it would positively impact the overall project.

## Testing Requirements

All tests may use `pytest` to ensure the code works before merging into the main project.

## Documentation Standards

Before each function, you must explain what it does, what input it is expecting, and what kind of output it produces. When documenting a pull request, be detailed in what it is fixing and/or adding to the project.

## Community Guidelines and Expectations

You can find the community guidelines in the [CODE_OF_CONDUCT.md](https://github.com/NAU-OSS/AutoWordle/blob/main/CODE_OF_CONDUCT.md)
