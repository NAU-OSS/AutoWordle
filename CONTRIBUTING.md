# Contributing

When contributing to this repository, please first discuss the change you wish to make via GitHub issue or email (ags392@nau.edu). 

## Pull Request Process

1. Ensure any install or build dependencies are removed before the end of the layer when doing a 
   build.
2. Update the README.md with details of changes to the interface, which includes new dependencies.
3. Increase the version numbers in any example files and the README.md to the new version that this
   Pull Request would represent. The versioning scheme we use is [SemVer](http://semver.org/).
4. You may merge the Pull Request once you have the sign-off from Aiden Seay.

## Reporting Bugs

Open a new issue within the GitHub issue tracker. Be descriptive and include steps on how to replicate the bug. Additionally, if you'd like, you may write a proposed solution.
